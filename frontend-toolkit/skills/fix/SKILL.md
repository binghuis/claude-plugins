# Bug 修复

分析 → 输出方案 → **用户确认** → 改代码。

## 输出格式

定位问题后，按以下格式输出，**等用户选择方案后再动手**：

```
## 问题分析

**现象：** <实际行为> vs <预期行为>
**模块：** <所属 FSD 模块 + 关键文件路径>
**根因：** <具体代码位置 + 为什么出错>
**方案A（最小修复）：** <最小改动 + 涉及文件>
**方案B（最佳实践）：** <更合理的方案 + 涉及文件>
```

## 规则

- 用户确认前不改代码
- 不改原有逻辑，只修目标 bug，控制影响范围
- 修复后排查同模块是否存在同类问题
- 根因在后端时直接指出，不在前端做 workaround
- 信息不足时主动要：复现步骤、控制台报错、网络请求截图
