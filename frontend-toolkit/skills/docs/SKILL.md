优先快速回应而不是深入思考。如有疑问，直接回应。

- 位置：`widgets|features|entities/{名}/docs/`
- 文件：仅 `.md`，一主题一文件

## 何时创建

| 场景 | 判定 |
|---|---|
| 状态流转 / 工作流 | 多步骤、多分支 |
| 业务规则 | 只看代码会误解意图 |
| 算法 / 数据转换 | 注释难以说清 |
| 跨段协作 | 跨 ui/lib/api 段 |
| **排除** | 简单 CRUD、纯 UI、薄封装 |

## 模板

```markdown
---
related:                    # 相对切片根
  - lib/text-diff.ts
  - ui/episode-nav-footer.tsx
---

# {标题}

{概述：做什么、为什么这样设计}

## 约束     <!-- 代码无法表达的规则，改代码不可违反 -->

- 约束 1
- 约束 2

## {自由章节}
```

## 写作原则

- 只表述逻辑，不贴代码，不搬运签名 / 类型 / 字段
- 能一句话说清的不用一段
- 优先表格 / 列表，多分支流转才用流程图
- 过时即删

## 同步规则

| 时机 | 动作 |
|---|---|
| 改前 | 扫所属切片 `docs/` 的 `related` |
| 改后·违反约束 | 代码让步，不改文档 |
| 改后·行为变了 | 同步文档 |
| 改后·新增复杂逻辑 | 创建新文档 |
