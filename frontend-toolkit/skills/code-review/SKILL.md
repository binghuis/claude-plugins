# Code Review

只找核心隐患 — 会导致业务异常或严重性能问题的代码。不报风格、命名、格式等边角问题。

## 变更来源

依次取第一个有内容的：
1. `git diff --cached`
2. `git diff`
3. `git diff HEAD~1`

需要时读取上下文文件辅助判断。

## 维度

1. **数据流** — 竞态条件、缓存与实际数据不一致、异步顺序依赖、状态更新丢失
2. **逻辑隐患** — 条件分支遗漏、逻辑不一致（同一概念多处判断标准不同）、默认值掩盖的空数据问题
3. **权限与状态** — 权限判断遗漏或不一致、角色/状态组合未覆盖、不同入口走了不同的权限逻辑
4. **设计问题** — 职责不清导致修改连锁反应、状态放置不当导致数据源不一致、组件耦合导致不可预测行为

## 输出

每个问题：维度 + 严重程度 + 原因 + 位置。

```
#### [标题] — [数据流|逻辑|权限|设计] 🔴
**原因**：...
**位置**：`file:line`
```

最后附总结：问题数 + Top 3 优先修复项。无问题直接说"无问题"。

## 规则

- 只报会出 bug 或严重影响性能的问题
- 只指出问题和原因，不提供改进代码
- 没问题不凑数
