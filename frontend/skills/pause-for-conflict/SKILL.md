在回答之前仔细思考并一步一步来；这个问题比看上去要难。

A = 我电脑上的代码；B = 外来的。与 git 的 ours/theirs 不直接对应。

| op | A | B |
|---|---|---|
| `git merge X`（在 Y 上） | Y | X |
| `git rebase Y`（在 X 上） | X | Y |
| `git pull` | 本地未推 commit | 拉来的 |
| `git cherry-pick X` | 当前分支已有 | X |

## 流程

1. `git status` 判 op，按上表标 A / B
2. 读两边代码、git log、changelog，每处一句话讲清做了什么 + 为什么
3. 每处给推荐选项 + 理由 + 代价；信息不足停下问用户，得答复再续
4. 按"呈现模板"把每处冲突输出给用户
5. 用 `AskUserQuestion` 问选择（C = 合并两边）：
   - 1 处冲突：`A / B / C`
   - 多处冲突：`全选 A / 全选 B / 全选 C / 自定义`；自定义等用户速记 `1A 2B 3C`
6. 按选择落盘，只改冲突标记内的代码

## 呈现模板

```
1. <文件路径> · <模块> — <分歧一句话>
   A（<分支/提交>）：<做了什么 + 为什么>
   B（<分支/提交>）：<做了什么 + 为什么>
   → 推荐 A/B/C：<理由>；代价：<丢什么>
```

## 边界

| 情况 | 处理 |
|---|---|
| 纯格式（空行 / 缩进）无语义差异 | 按项目规范直接合 |
| 用户说"你看着办" | 说选择 + 理由，再动手 |
