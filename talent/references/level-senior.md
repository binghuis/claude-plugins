# 高级(7+ 年)能力标准

通用维度与 proxy 信号见 [talent-model.md](./talent-model.md)。本文件只定义"在高级这一档,这些维度该用什么标准看"。

## 本级聚焦

- **主看**:架构判断事件、领域深度、守门人时刻
- **次看**:团队影响力 / 教练、跨域协作、AI 协作工程化深度
- **不该考**:此级别全维度都该考——**守门人事件和架构判断事件是不可替代的高分项**

## 核心问题

高级岗位最大的伪信号是**履历豪华**——大厂 title、长年限、参与过大项目,这些在 AI 时代红利期会非常常见,但**判断力、架构主导、守门人意识**这三件事没有捷径。

评估要倒过来:**默认存疑,找证据证明这个人真的做过决策、真的叫停过、真的为某个架构决定签过字**。

## 三大核心证据(缺任何一个都是显著降权)

### 1. 架构判断事件

不是"设计了 X 架构",是"**评估了 A、B、C 三个方案,选了 B 因为...,后来发现...,调整为...**"。

| ✓ 强证据 | ✗ 弱证据 |
|---|---|
| "评估 SSR / SSG / CSR,选 SSG 因为内容更新频率 X,SEO 收益 Y" | "采用 Next.js 实现 SSR" |
| "推翻原微前端方案,改为模块联邦,因为构建性能从 X 提升到 Y" | "搭建微前端架构" |
| "拒绝引入 Redux,理由是状态复杂度未达阈值,后续证明判断正确" | "使用 Redux 管理全局状态" |

### 2. 守门人时刻

| ✓ 强证据 | ✗ 弱证据 |
|---|---|
| "叫停某双十一上线方案,因为压测数据显示 X 风险" | "保障双十一稳定上线" |
| "推翻团队已工作 3 周的方案,因为发现 Y 不可行" | "推动方案落地" |
| "拒绝产品需求,因为用户调研显示...,改为 Z 方案" | "支持产品需求" |
| "下线 W 功能,因为维护成本远超收益" | "维护 W 功能" |

**关键**:守门人证据必须有**叫停 / 拒绝 / 推翻 / 下线**这类反向动作。只有"推动 / 落地 / 上线"的高级候选人,要存疑。

### 3. 领域深度外化

7+ 年从业,必然有外化产出。**没有外化产出的"深度",大概率不是真深度**。

| 类别 | 例 |
|---|---|
| 技术写作 | 持续更新的博客、深度长文、书籍 |
| 公开演讲 | 大会演讲、内部分享录像 |
| 开源贡献 | 自己的开源项目(不是 fork)、对主流项目的实质 PR |
| 标准/工具 | 制定过技术规范、贡献过工具链 |
| 培养 | 带过的人现在在做什么 |

## 正信号

| 信号 | 证据样态 |
|---|---|
| 架构判断事件 | 见上方"三大核心证据 1" |
| 守门人时刻 | 见上方"三大核心证据 2"——出现"叫停/推翻/拒绝/下线"动词 |
| 领域深度外化 | 见上方"三大核心证据 3" |
| 跨域协作 | 与后端/算法/产品/设计/运维有具体协作产出,而非各干各的 |
| 团队建设 | 培养出 X 名工程师晋升、推动了 Y 项工程文化变革 |
| AI 时代领导力 | 主导过团队 AI 工作流改造、有具体产出可衡量 |
| 长期一致性 | 多份工作中可以看出一致的判断风格和价值观 |

## 反信号

| 反信号 | 处理 |
|---|---|
| 履历豪华但全是"负责/推动/参与" | 降权 + 标问询——title 大未必判断弱,但要面试验 |
| 7+ 年但只有"做了什么",没有"为什么不做什么" | 强降权——决策证据缺失,这是高级核心维度 |
| 完全无叫停/推翻/拒绝/下线痕迹 | 强降权——守门人维度空白(三大核心证据之一) |
| 无任何外化产出(无博客/无演讲/无开源) | 标问询 + 降权——可能是低自我营销型,**面试必问"为什么不写"**,回答能服人不算反信号 |
| 大量 buzzword 堆砌 | 降权——表达水化,思考可能也水化 |
| 完全无 AI 协作工程化痕迹 | 强降权——2026 年的高级工程师对此无感是危险信号 |
| 跳槽周期短(多份 1 年以内)+ 无明确说明 | 标问询——文化轮验,不简历筛 |
| 项目数量极多(10+)但每个都很表层 | 降权——广而不深的高级版本 |
| 自述"精通 / 专家 / 资深"但具体描述配不上 | 降权——自我标签与证据不匹配 |

> **怎么用反信号**:单个反信号不构成 pass。**dealbreaker** = 三大核心证据(架构判断 / 守门人 / 领域外化)**全空** → 默认 pass;只缺一项 → 降权 + 反向验证要狠。

## 顶尖 5% 锚点(ceiling 锚,差异化 reason to hire)

高级 ceiling 不是"过线",是"在领域里有不可替代的位置"。顶尖锚点:

| 锚点 | 长什么样 |
|---|---|
| **定义过领域问题** | 不止解决问题,还**定义了什么是问题**——制定过技术规范 / 引领过技术方向 / 提出过被采纳的概念 |
| **多次叫停被验证正确** | 守门人不是"我拒绝过",而是"我叫停了 X,半年后确认风险是真的"——有事后验证的判断力 |
| **被同行追随** | 同行愿意 follow 他的判断;离职带走过工程师跟随;前同事还在用他制定的规范 |
| **培养出晋升的人** | 带过的人现在在做什么——有具体成长轨迹的 mentee,而不只是"带过 N 人" |
| **跨公司影响** | 影响力溢出当前公司——开源项目被多公司采用 / 演讲在大会被引用 / 有行业声誉 |
| **元判断稳定性** | 多份工作中能看到一致的判断风格:对什么坚持、对什么放手——不是机会主义 |

**ceiling 锚点是"为什么招他不招别人"的核心**——高级岗位最大的失败模式是"过 floor 但平庸",招进来就是更贵的执行者。**有 ceiling 锚点 ≠ 自动推荐**——还要看与团队当前缺什么匹配——但**没有任何 ceiling 锚点的高级,默认存疑**。

## 高级评估的特殊视角

- **倒过来读**:先看反信号,符合多项直接 pass,不浪费时间挖正信号。
- **GitHub 必看**:7+ 年还没有可看的 GitHub,基本是反信号。
- **质疑年限**:7 年中有 5 年在做相同层次的事,本质是 2 年经验重复 5 次。
- **看一致性**:多份工作的判断风格、决策密度、外化产出节奏是否一致。
