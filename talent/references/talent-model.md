# AI 时代程序员能力评价标准

## 一句话

打字快不再是优势。**让 AI 替你打字、并且打对、并且你敢签字**,才是。

---

## 一、能力清单

评价一个程序员,看这些维度:

- **问题定义** —— 把模糊需求切成可执行的具体问题;prompt 即规约
- **审阅与验证** —— 读懂代码(包括 AI 写的),嗅出 bug / 性能 / 安全坑;为代码留下测试、可观测、复盘
- **AI 协作力** —— 何时用、选什么模型、怎么提问、怎么校验、怎么把工作流编排进工程实践(hooks / skills / agents / 自动化)
- **守门人意识** —— AI 是放大器,会把好判断和坏判断同等放大。每次 commit / merge / 部署都是一次签字,不可外包
- **系统思维** —— 跨文件、跨服务、跨时间的全局视角
- **领域深度** —— 业务规则、技术栈底层、性能边界
- **学习能力 / 适应力** —— 识别什么值得学、用 AI 加速学新栈、知识不折旧
- **团队影响力 / 教练** —— 把个人判断变成团队规范、带新人融入 AI 工作流
- **基础功底** —— 不再是日常打字内容,但 debug AI 错误时是唯一兜底
- **判断力 / 品味**(伞盖)—— 由以上几项合成观察。表征:什么该做 / 不该做 / 何时停、架构取舍、抽象时机、命名直觉

## 二、Proxy 信号:怎么观察

能力不能直接测,只能通过证据反推。

### 问题定义
- ✅ 自己开题的项目(开源/内部工具);项目描述含"为什么做 / 为什么不做"
- ❌ 全是接需求做需求;项目动词只有"实现 / 开发 / 完成"

### 审阅与验证
- ✅ 主导/参与过 code review 流程、写过审查清单、线上事故复盘文章;测试 / 可观测建设痕迹
- ❌ 从未提及测试、监控、复盘

### AI 协作力
- ✅ 个人 hooks / skills / agents、自动化工作流、AI 工具深度使用痕迹、prompt 库
- ❌ 完全无 AI 工具痕迹(在 2026 年)

### 守门人意识
- ✅ "推翻了 X / 拒绝了 Y / 让 Z 停下来 / 降级了 W"事件
- ❌ 只有"推动 / 落地 / 上线",没有"叫停"

### 系统思维
- ✅ 跨服务、跨团队的项目角色;端到端拥有过模块
- ❌ 螺丝钉位、单点功能负责

### 领域深度
- ✅ 同领域 3+ 年 + 输出物(博客 / 演讲 / 开源 / 专利)
- ❌ 同领域 3+ 年但只有"参与"型描述

### 学习能力 / 适应力
- ✅ GitHub 提交 / 博客时间序列显示能力增速;快速迁移新栈的具体案例
- ❌ 技能栏停留在数年前版本("精通 React 16");只列已会、无学习痕迹

### 团队影响力 / 教练
- ✅ 主导团队规范 / 内部分享 / 写过 onboarding 文档 / mentee 成长可见 / 跨团队推动落地
- ❌ 多年高 P 但只有个人贡献,从未影响团队工作流

### 基础功底
- ✅ 能在没有 AI 时定位疑难 bug;能讲清底层原理而不只是 API
- ❌ 离开 AI 寸步难行;只会调用、不会拆解

### 判断力 / 品味(合成观察)
- ✅ 措辞含"取舍 / 拒绝 / 推翻 / 降级 / 简化 / 拆分"
- ❌ 措辞全是"使用 X 框架 / 集成 Y 库"
