# 初级(0-3 年)能力标准

通用维度与 proxy 信号见 [talent-model.md](./talent-model.md)。本文件只定义"在初级这一档,这些维度该用什么标准看"。

## 本级聚焦

- **主看**:基础功底、学习能力 / 适应力、AI 协作力
- **次看**:问题分解萌芽
- **不该考**:架构判断、领域深度、守门人事件、团队影响力(看到算加分,看不到不扣分)

## 核心倒置

AI 吃掉了"练手区"(CRUD、样板、调 API 这些过去用来积累手感的活),初级的"已具备能力"本就稀缺。**学习曲线 > 当前水位**——一个能在 18 个月内从零做到能写完整功能的人,比一个三年了仍只会调 API 的人,在初级岗位上是更优解。判断力只能从大量实战中长出来,所以重点看**学习曲线证据**(自驱产出、深度作品、可见的能力增速),而不是简历列了多少项目。

## 正信号

| 信号 | 证据样态 |
|---|---|
| 基础功底扎实 | 对算法/数据结构/HTTP/浏览器原理的理解,而非只有框架名 |
| 自驱产出 | 个人项目/开源贡献/技术博客/比赛作品——任意一项都比"参与公司项目"重 |
| 学习曲线 | 时间序上能看到能力跃迁(课程作业 → 个人项目 → 实习产出深度递增) |
| AI 协作力萌芽 | 用 AI 工具完成过具体工作流(不是"会用 ChatGPT"这种描述) |
| 一个深度作品 | 一个能展开讲 30 分钟的项目,胜过 8 个 2 行项目 |
| 复现 / 阅读源码 | 读过 React/Vue/Webpack 等源码、复现过经典算法/系统 |

## 反信号

| 反信号 | 处理 |
|---|---|
| 技能栏列 10+ 框架/语言"精通/熟练" | 标问询——AI 时代前的简历模板,不一定能力差 |
| 项目栏堆砌 6+ 课程作业,每个 2 行 | 降权(若同时无深度作品)——广而不深 |
| 项目描述全是"使用 X / 集成 Y / 开发 Z" | 标问询——纯执行表象,可能只是不会写简历 |
| 完全无 AI 工具使用痕迹 | 标问询——2026 年的初级,这个空白需要解释 |
| 没有任何 GitHub / 博客 / 作品链接 | 降权——高带宽信号源缺失 |
| 简历精致但 GitHub 空白 | 降权——包装思维线索,需面试印证 |
| 全是大厂"参与/协助"实习,无端到端产出 | 标问询——确认是否能展开讲细节 |

> **怎么用反信号**:单个不降档,只标"面试该追问的洞";3+ 同时出现且无正信号兜底才降权。

## 顶尖 5% 锚点(ceiling 锚,差异化 reason to hire)

floor 是"达标",ceiling 是"为什么招他不招别人"。初级顶尖锚点:

| 锚点 | 长什么样 |
|---|---|
| **自驱产出有真用户** | 开源项目有 star / fork 不止自己;博客被人长期阅读;比赛/作品有外部反馈 |
| **学习曲线极陡** | 6-12 个月内从零掌握新栈并产出深度作品;时间序看到能力**加速度**而不只是积累 |
| **判断萌芽** | 能讲清"我为什么没用 X / 我尝试了 A 但选了 B"——同年限里这是稀有信号 |
| **底层好奇心** | 主动读源码 / 写过工具链贡献 / 复现过经典论文或系统——不止用,还想懂 |
| **AI 协作早熟** | 不只是"用过",而是有自己的工作流 / prompt 库 / 自动化痕迹 |

**没有 ceiling 锚点不扣分**(初级本来稀缺),但有任意一条 → 是"差异化 reason to hire",不是简单的"过 floor"。
