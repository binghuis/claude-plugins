# 中级(3-7 年)能力标准

通用维度与 proxy 信号见 [talent-model.md](./talent-model.md)。本文件只定义"在中级这一档,这些维度该用什么标准看"。

## 本级聚焦

- **主看**:问题定义、审阅证据(含工程素养)、跨模块系统思维
- **次看**:AI 协作工程化、领域知识积累、学习能力外化
- **不该考**:端到端架构主导、深度团队影响力(看到算超规格加分,看不到不扣分)

## 核心问题

中级是分水岭——往上是判断者,往下是执行者。同样是"3 年 React 经验",一个人在主导模块设计,另一个在按图施工,看起来差不多,**关键差别在动词和因果**。

中级也是最容易看走眼的一档:既不能用初级的"潜力"逻辑,也不能用高级的"事件"逻辑——必须看**已经发生的、跨模块的、有工程素养痕迹的**实际产出。

## 动词扫描(核心方法)

中级评估最有效的单一信号是**项目描述的动词分布**。

| 类别 | 动词 | 含义 |
|---|---|---|
| 判断性(高分) | 取舍 / 拒绝 / 推翻 / 降级 / 简化 / 拆分 / 合并 / 重构 / 替换 | 候选人在做决策 |
| 执行性(中性) | 实现 / 开发 / 完成 / 集成 / 联调 | 候选人在做事 |
| 模糊性(低分) | 负责 / 参与 / 协助 / 推动 / 赋能 | 角色不清,需追问 |

**经验法则**:
- 判断性动词占比 ≥ 30%——强信号
- 全是执行性动词——要追问"具体决策权在谁手里"
- 大量模糊性动词——降权,这是简历包装术

## 正信号

| 信号 | 证据样态 |
|---|---|
| 问题定义能力 | "发现 X 问题→定义为 Y→给出方案 Z"的因果链;提出过被采纳的技术方案 |
| 审阅证据 | 主导/搭建过 code review 流程;写过审查清单/规范;线上事故复盘文章 |
| 跨模块系统思维 | 端到端拥有过模块(从设计 → 开发 → 上线 → 监控);跨服务集成经验 |
| AI 协作工程化 | 不是"会用 AI",而是"把 AI 编排进了某个工作流并产生了具体收益" |
| 技术写作 | 博客/演讲/内部分享——证明能把判断说清楚 |
| 性能/质量数字 | "首屏从 X 降到 Y / 错误率从 A 降到 B"——带数据的成果 |

## 反信号

| 反信号 | 处理 |
|---|---|
| 履历豪华但全是"负责/参与"型描述 | 降权 + 标问询——3+ 年仍是螺丝钉,但也可能只是不会写简历 |
| 技能栏堆砌 + 全是框架名,无任何深度领域 | 标问询——表层游可能,也可能简历模板老派 |
| 项目数量 ≥ 8 但每个深度浅 | 降权——广而不深的强信号 |
| 完全没有线上事故 / 复盘 / 监控痕迹 | 降权——审阅与验证维度空白 |
| 完全无 AI 协作工程化痕迹 | 标问询——2026 年的中级,这是显著缺位 |
| 项目描述全是技术名词,无业务理解 | 标问询——领域深度可能在但没写出来 |
| GitHub 空白或全是 fork | 降权——3+ 年无沉淀,但需面试印证(可能在公司内网) |
| 通篇 buzzword("赋能/抓手/闭环") | 降权——表达浅或思考浅 |

> **怎么用反信号**:单个不降档,只标"面试该追问的洞";3+ 同时出现且无正信号兜底才降权。**dealbreaker** = 完全无判断性动词 + 完全无审阅/复盘痕迹 → 已是高级执行者,招进来也长不出来。

## 顶尖 5% 锚点(ceiling 锚,差异化 reason to hire)

floor 是"决策证据达标",ceiling 是"判断能力领先同年限"。中级顶尖锚点:

| 锚点 | 长什么样 |
|---|---|
| **元判断**(推翻自己) | 不止"我推翻了 X",还有"我推翻了**自己**的 Y——后来发现初判错在哪" |
| **被复用的产出** | 写过的工具 / 规范 / 文档被团队外的人采用;开源项目有用户;PR 被主流项目合并 |
| **跨域改造工作流** | 跨域协作不只是配合,而是**改变了对方的工作流**(让 PM/QA/SRE 工作方式变了) |
| **审阅深度** | 主导过 review 文化建设,而不只是参与;事故复盘有可复用的方法论沉淀 |
| **AI 协作工程化深度** | 不只是用 AI,还**评估并选择了**特定模型 / 工作流的取舍,有数据支撑 |
| **技术写作有判断锐度** | 博客 / 分享不是教程汇编,而是带"为什么不做 X"的论证 |

**没有 ceiling 锚点不直接扣分**(中级本就是分水岭),但有 → 是判断者倾向,不是高级执行者。
