## 输入

| 必填 | 候选人 PDF 路径(或拖拽进对话) |
|---|---|
| 选填 | 候选人级别(初 / 中 / 高,覆盖自动推断) |

## 流程

| 步骤 | 动作 |
|---|---|
| 1. 读 PDF + 识别姓名 | 用 Read 工具读 PDF;从内容推断姓名(推断不出问用户) |
| 2. 推断级别 | 见下方"定级速判";若用户已传级别,跳过推断 |
| 3. 加载本级标准 | 按级别选用对应 level-*.md(已通过 @ 引入,无需 Read) |
| 4. 信号挖掘 | 在 PDF 里找本级正/反信号,引用原文 |
| 5. 出 floor 判断 | **过 floor / 边缘 / 不过 floor**——见下方"floor 判断口径" |
| 6. 输出 | 按下方格式 |

## 定级速判(不是只看年限)

AI 时代里 7 年仍是高级执行者很常见,3 年成判断者也存在。**关键看判断证据密度**。

| 级别 | 关键信号 | 关键反信号 |
|---|---|---|
| 初级 | 0-3 年 + 基础功底 + 自驱产出 | 项目堆砌、零 GitHub、技能栏堆砌 |
| 中级 | 2-7 年 + **决策性动词**(取舍/拒绝/推翻/降级/简化)+ 跨模块 | 全是"负责/参与/协助"、纯执行型 |
| 高级 | 6+ 年 + 三大核心证据线索(架构判断 + 守门人 + 领域外化) | 履历豪华但"推动/落地"、无外化产出 |

**边界处理**:
- 年限符合 X 级 + 反信号显著 → 降一级口径
- 年限不到 + 判断证据强 → **可上推一级**(AI 时代常见)
- 年限远超 + 只到下一级标准 → 标"履历真实度疑点,反向验证要狠",用低级标准评估

## Floor 判断口径

只 3 档,不要"强烈推荐"这种 ceiling 表态——简历不是 ceiling 评估的素材。

| 档位 | 含义 | 触发条件 |
|---|---|---|
| **过 floor** | 值得花一场面试去验 | 命中本级主看维度的至少一个正信号 + 没有触发 dealbreaker |
| **边缘** | 值得面但有显著疑点,prep 时反向验证要狠 | 正信号弱 / 反信号成簇(3+) / 履历真实度疑点 |
| **不过 floor** | 不值得用一场面试去验 | dealbreaker 触发(见下) / 主看维度全空 / 反信号 5+ 且无正信号兜底 |

**Dealbreaker(直接不过 floor)**:
- 中级且**完全无任何**判断性动词 + 无任何审阅/复盘/系统设计痕迹 → 已是高级执行者,招进来也长不出来
- 高级且**完全无外化产出 + 完全无叫停/推翻/拒绝/下线动词 + 完全无架构方案对比** → 三大核心证据全空,默认 pass
- 简历事实**明显造假**(年限自相矛盾、技术栈与年代不符、项目时间穿越)

**单个反信号不构成 dealbreaker**——反信号是面试该追问的洞,不是 gate。3+ 反信号成簇才有意义。

## PDF 解析重点(各级共通)

| 段落 | 看什么 |
|---|---|
| 教育 | 学校层级仅作背景信号,不作录用依据;GPA 不参考 |
| 工作经历 | **判断性动词** vs 执行性动词的比例;有没有"为什么这么做 / 为什么不那么做"的因果链 |
| 项目 | **深度 > 数量**——一个能展开 30 分钟的项目胜过 8 个 2 行项目 |
| 个人项目 / 开源 | 高权重信号源(初级尤其稀缺) |
| 技能栏 | 重点扫反信号("精通/熟练"罗列、停留在数年前版本) |
| 技术写作 | 博客 / 演讲 / 分享——中级开始是高带宽证据,高级是必备 |
| 链接 | GitHub / 博客 / 演讲——**列出但不主动 fetch**,标记"建议人工查阅"+ 给出查阅理由 |

## 输出格式

```
## 候选人:[姓名/文件名]

### 级别推断:[初 / 中 / 高]
**依据**:[年限 + 判断证据密度;若用户已传级别,标"用户指定"]
**边界提醒**(若有):[如"履历真实度疑点 / 上推一级"]

### Floor 判断:[过 floor / 边缘 / 不过 floor]
**触发**:[过 → 哪条主看正信号 / 边缘 → 哪些反信号成簇 / 不过 → 哪条 dealbreaker]

### 命中正信号
- [信号]:[PDF 原文引用,加路径锚]
- ...

### 命中反信号
- [反信号]:[PDF 原文引用]
- ...

### 本级专属观察
按级别选填:
- 初级:**学习曲线评估** —— 能力增速是上升 / 平台 / 停滞?依据是什么
- 中级:**动词分布扫描** —— 判断性 / 执行性 / 模糊性动词比例,一句话评估
- 高级:**三大核心证据** —— 架构判断 / 守门人时刻 / 领域深度外化,逐一列证据或明写"未发现"

### 必看链接(建议人工核实)
- GitHub:[URL] —— 建议看的原因(项目深度 / commit 模式 / 个人项目 vs fork)
- 博客 / 演讲:[URL] —— 建议看的原因(深度 / 一致性 / 更新频率)

### 面试重点(PDF 看不出,需问)
- [基于本级"次看"或反信号留下的空白]
- ...

### 一句话判断
[本人值不值得用一场面试去验,为什么——**只回答 floor,不评 ceiling**]
```

## 硬规则

- **只判 floor,不评 ceiling**——不出"强烈推荐 / 推荐"这种 ceiling 表态;那是 hiring committee 的事
- **任何信号判断都必须引用 PDF 原文**,不引用不下结论
- **链接只列不点**——守门人不可外包
- **没有写在 PDF 里的能力,不替候选人脑补**;标记为"待面试验证"
- **单个反信号不降档**——记入"面试重点"待追问;3+ 反信号成簇才考虑"边缘";触发 dealbreaker 才"不过 floor"
- **高级三大核心证据全空 → 不过 floor**(默认 pass);只缺一项 → 边缘 + prep 反向验证要狠
- **中级完全无判断性动词 + 无审阅痕迹 → 不过 floor**;只是模糊性动词多 → 边缘 + 标"高级执行者倾向待验"
- **初级看不到学习曲线证据**——降权到边缘,但不直接 pass(初级正信号本就稀缺)
- **姓名识别失败必须问用户**——不能用 "候选人" / "unknown" 占位
