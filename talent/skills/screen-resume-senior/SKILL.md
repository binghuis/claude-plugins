在回答之前仔细思考并一步一步来；这个问题比看上去要难。

评判标准引用 @../talent-model/SKILL.md。本 skill 只负责"PDF 这个输入里怎么挖高级独有的信号"。

## 输入

用户提供 PDF 路径（或拖拽进对话）。Read 工具直接读取 PDF。

## 高级筛选的核心问题

高级岗位最大的伪信号是**履历豪华**——大厂 title、长年限、参与过大项目,这些在 AI 时代红利期会非常常见,但**判断力、架构主导、守门人意识**这三件事没有捷径。

筛选要倒过来：**默认存疑,找证据证明这个人真的做过决策、真的叫停过、真的为某个架构决定签过字**。

## PDF 解析重点

| 段落 | 看什么 |
|---|---|
| 工作经历 | 不看 title,看**架构判断事件**和**守门人时刻**的具体描述 |
| 项目描述 | "为什么不做 X"的论述比"做了 X"更值钱 |
| 领域深度外化 | 博客/演讲/开源/专利/书籍/标准制定——高级必须有外化产出 |
| 团队影响力 | 带过的人、培养出的人、推动过的工程文化变革 |
| 链接 | GitHub / 博客 / 演讲——列出 + 标记建议查阅理由 |

## 三大核心证据（缺任何一个都是显著降权）

### 1. 架构判断事件

不是"设计了 X 架构",是"**评估了 A、B、C 三个方案,选了 B 因为...,后来发现...,调整为...**"。

| ✓ 强证据 | ✗ 弱证据 |
|---|---|
| "评估 SSR / SSG / CSR,选 SSG 因为内容更新频率 X,SEO 收益 Y" | "采用 Next.js 实现 SSR" |
| "推翻原微前端方案,改为模块联邦,因为构建性能从 X 提升到 Y" | "搭建微前端架构" |
| "拒绝引入 Redux,理由是状态复杂度未达阈值,后续证明判断正确" | "使用 Redux 管理全局状态" |

### 2. 守门人时刻

| ✓ 强证据 | ✗ 弱证据 |
|---|---|
| "叫停某双十一上线方案,因为压测数据显示 X 风险" | "保障双十一稳定上线" |
| "推翻团队已工作 3 周的方案,因为发现 Y 不可行" | "推动方案落地" |
| "拒绝产品需求,因为用户调研显示...,改为 Z 方案" | "支持产品需求" |
| "下线 W 功能,因为维护成本远超收益" | "维护 W 功能" |

**关键**:守门人证据必须有**叫停 / 拒绝 / 推翻 / 下线**这类反向动作。只有"推动 / 落地 / 上线"的高级候选人,要存疑。

### 3. 领域深度外化

7+ 年从业,必然有外化产出。**没有外化产出的"深度",大概率不是真深度**。

| 类别 | 例 |
|---|---|
| 技术写作 | 持续更新的博客、深度长文、书籍 |
| 公开演讲 | 大会演讲、内部分享录像 |
| 开源贡献 | 自己的开源项目（不是 fork）、对主流项目的实质 PR |
| 标准/工具 | 制定过技术规范、贡献过工具链 |
| 培养 | 带过的人现在在做什么 |

## 高级正信号

| 信号 | 在 PDF 里的样子 |
|---|---|
| 架构判断事件 | 见上方"三大核心证据 1" |
| 守门人时刻 | 见上方"三大核心证据 2"——出现"叫停/推翻/拒绝/下线"动词 |
| 领域深度外化 | 见上方"三大核心证据 3" |
| 跨域协作 | 与后端/算法/产品/设计/运维有具体协作产出,而非各干各的 |
| 团队建设 | 培养出 X 名工程师晋升、推动了 Y 项工程文化变革 |
| AI 时代领导力 | 主导过团队 AI 工作流改造、有具体产出可衡量 |
| 长期一致性 | 多份工作中可以看出一致的判断风格和价值观 |

## 高级反信号

| 反信号 | 处理 |
|---|---|
| 履历豪华但全是"负责/推动/参与" | 强降权——title 大,实质判断证据空白 |
| 7+ 年但只有"做了什么",没有"为什么不做什么" | 强降权——决策证据缺失 |
| 完全无叫停/推翻/拒绝/下线痕迹 | 强降权——守门人维度空白 |
| 无任何外化产出（无博客/无演讲/无开源） | 强降权——深度无法验证 |
| 大量 buzzword 堆砌（"赋能/抓手/闭环/中台/数智化"） | 降权——表达水化,思考可能也水化 |
| 完全无 AI 协作工程化痕迹 | 强降权——2026 年的高级工程师对此无感是危险信号 |
| 跳槽周期短（多份 1 年以内）+ 无明确说明 | 标记问询 |
| 项目数量极多（10+）但每个都很表层 | 降权——广而不深的高级版本 |
| 自述"精通 / 专家 / 资深"但具体描述配不上 | 降权——自我标签与证据不匹配 |

## 高级筛选的特殊视角

- **倒过来读**:先看反信号,符合多项直接 pass,不浪费时间挖正信号。
- **GitHub 必看**（标记给人）:7+ 年还没有可看的 GitHub,基本是反信号。
- **质疑年限**:7 年中有 5 年在做相同层次的事,本质是 2 年经验重复 5 次。
- **看一致性**:多份工作的判断风格、决策密度、外化产出节奏是否一致。

## 输出格式

```
## 候选人：[姓名/文件名]

### 推荐档：[强烈推荐 / 推荐 / 待面试验证 / 不推荐 / 直接 pass]

### 三大核心证据
1. 架构判断事件：[列举,引用 PDF 原文；没有就明说"未发现"]
2. 守门人时刻：[列举,引用原文；没有就明说]
3. 领域深度外化：[博客/演讲/开源/专利,带链接]

### 命中正信号
- [信号]：[PDF 原文引用]
- ...

### 命中反信号
- [反信号]：[PDF 原文引用]
- ...

### 履历真实度评估
[年限 vs 判断密度是否匹配 / 是否一致／是否有"2 年经验重复 N 次"嫌疑]

### 必看链接（建议人工核实）
- GitHub：[URL] —— 建议看的原因（个人项目质量 / commit 模式 / star 项目）
- 博客 / 演讲：[URL] —— 建议看的原因（深度 / 一致性 / 更新频率）

### 面试重点（PDF 看不出,需问）
- 一次推翻自己 / 团队方案的经历
- 一次叫停项目的经历
- 当前最不擅长的领域是什么（自我认知）
- AI 时代对所在技术领域的判断
- ...

### 一句话判断
[判断者 / 决策者级别,还是高级执行者；为什么；是否值得高级岗面试]
```

## 硬规则

- 不在筛选阶段做"招聘决定",只输出"是否值得面试"
- 三大核心证据缺任何一项,默认档位下调一档
- 没有外化产出的"深度"自述,不采信
- 任何判断都必须引用 PDF 原文,不引用不下结论
- 链接只列不点——守门人不可外包
- 高级筛选默认存疑,反信号优先扫描
