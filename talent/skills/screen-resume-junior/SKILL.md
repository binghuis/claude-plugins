在回答之前仔细思考并一步一步来；这个问题比看上去要难。

评判标准引用 @../talent-model/SKILL.md。本 skill 只负责"PDF 这个输入里怎么挖初级独有的信号"。

## 输入

用户提供 PDF 路径（或拖拽进对话）。Read 工具直接读取 PDF。

## 初级筛选的核心倒置

AI 吃掉了练手区,初级的"已具备能力"本就稀缺。**学习曲线 > 当前水位**。一个能在 18 个月内从零做到能写完整功能的人,比一个三年了仍只会调 API 的人,在初级岗位上是更优解。

## PDF 解析重点

| 段落 | 看什么 |
|---|---|
| 教育 | 学校层级仅作背景信号,不作录用依据；GPA 不参考 |
| 实习/项目 | **深度 > 数量**：深度看一个项目能展开多少细节、做了什么取舍 |
| 个人项目 / 开源 | 高权重——这是初级最稀缺的"自驱产出"信号 |
| 技能栏 | 重点扫**反信号**（见下） |
| 链接 | GitHub / 博客 / 个人站——列出但不主动 fetch,标记"建议人工查阅"+ 给出查阅理由 |

## 初级正信号（在 PDF 里挖什么）

| 信号 | 在 PDF 里的样子 |
|---|---|
| 基础功底扎实 | 描述里能看到对算法/数据结构/HTTP/浏览器原理的理解,而非只有框架名 |
| 自驱产出 | 个人项目/开源贡献/技术博客/比赛作品——任意一项都比"参与公司项目"重 |
| 学习曲线 | 时间序上能看到能力跃迁（从课程作业 → 个人项目 → 实习产出深度递增） |
| AI 协作力萌芽 | 提到用 AI 工具完成过具体工作流（不是"会用 ChatGPT"这种描述） |
| 一个深度作品 | 一个能展开讲 30 分钟的项目,胜过 8 个 2 行项目 |
| 复现 / 阅读源码 | 提到读过 React/Vue/Webpack 等源码、复现过经典算法/系统 |

## 初级反信号（看到要降权或淘汰）

| 反信号 | 处理 |
|---|---|
| 技能栏列 10+ 框架/语言"精通/熟练" | 强降权——这是 AI 时代前的简历模板 |
| 项目栏堆砌 6+ 课程作业,每个 2 行 | 强降权——广而不深 |
| 项目描述全是"使用 X / 集成 Y / 开发 Z" | 降权——纯执行,无判断萌芽 |
| 完全无 AI 工具使用痕迹 | 标记问询——2026 年的初级,这个空白需要解释 |
| 没有任何 GitHub / 博客 / 作品链接 | 降权——高带宽信号源缺失 |
| 简历精致但 GitHub 空白（点开看了的话） | 强降权——包装思维 |
| 全是大厂"参与/协助"实习,无端到端产出 | 不是淘汰项,但要确认是否能展开讲细节 |

## 初级不该看的（这级别不要求）

- 架构判断事件
- 跨服务/跨团队主导经验
- 领域深度（3+ 年沉淀）
- 守门人时刻

筛初级时看到这些只算加分,**没有不应扣分**。

## 输出格式

```
## 候选人：[姓名/文件名]

### 推荐档：[强烈推荐 / 推荐 / 待面试验证 / 不推荐]

### 命中正信号
- [信号]：[PDF 原文引用,加路径锚]
- ...

### 命中反信号
- [反信号]：[PDF 原文引用]
- ...

### 学习曲线评估
[一两句话：能力增速是上升、平台、还是停滞？依据是什么]

### 必看链接（建议人工核实）
- GitHub：[URL] —— 建议看的原因（项目深度 / commit 频率 / 个人项目 vs fork）
- 博客：[URL] —— 建议看的原因（技术深度 / 思考密度）

### 面试重点（PDF 看不出,需问）
- AI 协作力的具体工作流
- [深度作品名] 的取舍细节
- ...

### 一句话判断
[本人是否值得进入面试,为什么]
```

## 硬规则

- 不在筛选阶段做"招聘决定",只输出"是否值得面试"
- 任何反信号判断都必须引用 PDF 原文,不引用不下结论
- 链接**只列不点**——守门人不可外包
- 没有写在 PDF 里的能力,不替候选人脑补；标记为"待面试验证"
