在回答之前仔细思考并一步一步来；这个问题比看上去要难。

## 定位

这是评判标准本体（**WHO**——什么是好的工程师），不承担任何场景化执行（**HOW**——简历怎么筛、面试怎么问）。被场景类 skill 引用。

底层是判断力,中层是问题定义与审阅,上层是 AI 协作工程化,贯穿三层是**不可外包的守门人责任**。

## 三档定价

### ↑ 升值（AI 越强越值钱）

| 能力 | 含义 |
|---|---|
| 问题定义 | 把模糊需求切成可执行的具体问题；prompt 即规约 |
| 审阅与验证 | 读懂代码（包括 AI 写的），嗅出 bug / 性能 / 安全坑 |
| 判断力 / 品味 | 什么该做、什么不该做、什么时候停；架构取舍、抽象时机、命名直觉 |
| 系统思维 | 跨文件、跨服务、跨时间的全局视角 |
| 领域深度 | 业务规则、技术栈底层、性能边界 |

### = 重要但形式变了

| 能力 | 变化 |
|---|---|
| 调试定位 | 从"修自己的 bug"→"修 AI 写的 bug"；难度上升,因为缺少写代码时的心智模型 |
| 沟通对齐 | 产出变快,对齐反而更高频 |
| 基础功底 | 不再是日常打字内容,但 debug AI 错误时是唯一兜底 |

### ↓ 贬值（不再是壁垒）

| 类别 | 例 |
|---|---|
| 死记 | API、语法、框架细节 |
| 体力活 | 模板代码、CRUD、胶水代码、八股文式实现 |
| 速度 | 单纯打字快 |

## 新增元能力

| 能力 | 含义 |
|---|---|
| AI 协作力 | 何时用、选什么模型、怎么提问、怎么校验、怎么把工作流编排进工程实践（hooks / skills / agents / 自动化）。这是**新工种素养**,不是工具技能 |
| 守门人意识 | AI 是放大器,会把好判断和坏判断**同等**放大。每次 commit / merge / 部署都是一次签字——**不可外包** |

## 级别画像

各级看的能力本质不同,不是程度差异。

| 级别 | 主看 | 次看 | 不看（这级不该考） |
|---|---|---|---|
| 初级 | 基础功底、学习曲线证据、AI 协作力 | 问题分解能力萌芽 | 架构判断、领域深度、守门人事件 |
| 中级 | 问题定义、审阅证据、跨模块系统思维 | AI 协作工程化、领域知识积累 | 端到端架构主导 |
| 高级 | 架构判断事件、领域深度、守门人时刻 | 团队影响力、跨域协作 | （此级别全维度都该考） |

**初级注意**:AI 吃掉了"练手区"（CRUD、样板、调 API 这些过去用来积累手感的活）。判断力只能从大量实战中长出来——所以初级要看**学习曲线证据**（自驱产出、深度作品、可见的能力增速）,而不是简历列了多少项目。

## Proxy 信号（每条能力的可观察证据）

| 能力 | 正信号 | 反信号 |
|---|---|---|
| 问题定义 | 自己开题的项目（开源/内部工具）；项目描述含"为什么做 / 为什么不做" | 全是接需求做需求；项目动词只有"实现 / 开发 / 完成" |
| 审阅与验证 | 主导/参与过 code review 流程、写过审查清单、有线上事故复盘文章 | 从未提及测试、监控、复盘 |
| 判断力 / 品味 | 措辞含"取舍 / 拒绝 / 推翻 / 降级 / 简化 / 拆分" | 措辞全是"使用 X 框架 / 集成 Y 库" |
| 系统思维 | 跨服务、跨团队的项目角色；端到端拥有过模块 | 螺丝钉位、单点功能负责 |
| 领域深度 | 同领域 3+ 年 + 输出物（博客 / 演讲 / 开源 / 专利） | 同领域 3+ 年但只有"参与"型描述 |
| 调试定位 | 复杂线上事故的复盘、性能优化案例（带数据） | 没有任何故障/排障痕迹 |
| AI 协作力 | 个人 hooks/skills/agents、自动化工作流、AI 工具深度使用痕迹、prompt 库 | 完全无 AI 工具痕迹（在 2026 年） |
| 守门人意识 | "推翻了 X / 拒绝了 Y / 让 Z 停下来 / 降级了 W"事件 | 只有"推动 / 落地 / 上线",没有"叫停" |

## 通用反信号清单

| 信号 | 为什么是反信号 |
|---|---|
| "精通 X 语言 / 熟练 Y 框架"罗列式技能栏 | AI 时代这种自我定位本身可疑;"精通"是 1990 年代的简历语言 |
| 项目数量 ≫ 项目深度（罗列 8+ 项目,每个 2 行） | 广而不深,练手区思维 |
| 项目动词全是"实现了 / 开发了 / 完成了" | 没有判断、取舍、拒绝的痕迹,执行者而非决策者 |
| 履历全是大厂螺丝钉位,从未端到端拥有任何东西 | 缺系统思维证据 |
| 完全无 AI 工具使用痕迹 | 在 2026 年是显著反信号（除非候选人解释清楚） |
| 简历包装精致但 GitHub / 博客空白 | 高带宽信号源缺失,只有低带宽自述 |
| 通篇 buzzword（"赋能 / 抓手 / 闭环"或技术黑话堆砌） | 表达能力差或思考浅 |

## 结构性后果

**初级岗位更难做,中高级更稀缺**。AI 吃掉了练手区,但判断力只能从实战长出——成为资深的路径被拉长了,而对已经资深的人,杠杆变大了。

筛选时要意识到:对初级,正信号本就稀缺,**学习曲线**比"已具备能力"更重要;对高级,**守门人事件**和**架构判断事件**是不可替代的高分项。

## 一句话

打字快不再是优势,**让 AI 替你打字、并且打对、并且你敢签字**,才是。
