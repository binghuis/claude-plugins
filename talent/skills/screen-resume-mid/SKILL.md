在回答之前仔细思考并一步一步来；这个问题比看上去要难。

评判标准引用 @../talent-model/SKILL.md。本 skill 只负责"PDF 这个输入里怎么挖中级独有的信号"。

## 输入

用户提供 PDF 路径（或拖拽进对话）。Read 工具直接读取 PDF。

## 中级筛选的核心问题

中级是分水岭——往上是判断者,往下是执行者。同样是"3 年 React 经验",一个人在主导模块设计,另一个在按图施工,简历上看起来差不多,**关键差别在动词和因果**。

## PDF 解析重点

| 段落 | 看什么 |
|---|---|
| 工作经历 | **判断性动词** vs 执行性动词的比例（见下） |
| 项目描述 | 有没有"为什么这么做 / 为什么不那么做"的因果链 |
| 跨模块协作 | 端到端拥有过什么；跨服务/跨团队角色 |
| 技术博客 / 分享 | 中级开始,博客/分享是真实判断力的高带宽证据 |
| 链接 | GitHub / 博客 / 演讲——列出 + 标记建议查阅理由 |

## 动词扫描（核心方法）

中级筛选最有效的单一信号是**项目描述的动词分布**。

| 类别 | 动词 | 含义 |
|---|---|---|
| 判断性（高分） | 取舍 / 拒绝 / 推翻 / 降级 / 简化 / 拆分 / 合并 / 重构 / 替换 | 候选人在做决策 |
| 执行性（中性） | 实现 / 开发 / 完成 / 集成 / 联调 | 候选人在做事 |
| 模糊性（低分） | 负责 / 参与 / 协助 / 推动 / 赋能 | 角色不清,需追问 |

**经验法则**:
- 判断性动词占比 ≥ 30%——强信号
- 全是执行性动词——要追问"具体决策权在谁手里"
- 大量模糊性动词——降权,这是简历包装术

## 中级正信号

| 信号 | 在 PDF 里的样子 |
|---|---|
| 问题定义能力 | "发现 X 问题→定义为 Y→给出方案 Z"的因果链；提出过被采纳的技术方案 |
| 审阅证据 | 主导/搭建过 code review 流程；写过审查清单/规范；线上事故复盘文章 |
| 跨模块系统思维 | 端到端拥有过模块（从设计 → 开发 → 上线 → 监控）；跨服务集成经验 |
| AI 协作工程化 | 不是"会用 AI",而是"把 AI 编排进了某个工作流并产生了具体收益" |
| 技术写作 | 博客/演讲/内部分享——证明能把判断说清楚 |
| 性能/质量数字 | "首屏从 X 降到 Y / 错误率从 A 降到 B"——带数据的成果 |

## 中级反信号

| 反信号 | 处理 |
|---|---|
| 履历豪华但全是"负责/参与"型描述 | 强降权——3+ 年仍是螺丝钉 |
| 技能栏堆砌 + 全是框架名,无任何深度领域 | 降权——3 年了还在表层游 |
| 项目数量 ≥ 8 但每个深度浅 | 降权——广而不深 |
| 完全没有线上事故 / 复盘 / 监控痕迹 | 降权——审阅与验证维度空白 |
| 完全无 AI 协作工程化痕迹 | 标记问询——2026 年的中级,这是显著缺位 |
| 项目描述全是技术名词,无业务理解 | 降权——领域深度缺失 |
| GitHub 空白或全是 fork | 强降权——3+ 年无沉淀 |
| 通篇 buzzword（"赋能/抓手/闭环"） | 降权——表达浅或思考浅 |

## 中级不该看的

- 架构主导（团队/部门级）
- 守门人时刻里的"叫停 X 业务"级别决策
- 跨域影响力

这些是高级标准,中级看到算超规格加分,看不到不扣分。

## 输出格式

```
## 候选人：[姓名/文件名]

### 推荐档：[强烈推荐 / 推荐 / 待面试验证 / 不推荐]

### 动词分布扫描
- 判断性动词：[列举,占比]
- 执行性动词：[列举,占比]
- 模糊性动词：[列举,占比]
- 评估：[一句话]

### 命中正信号
- [信号]：[PDF 原文引用]
- ...

### 命中反信号
- [反信号]：[PDF 原文引用]
- ...

### 系统思维证据
[端到端拥有过什么 / 跨模块协作的具体例子；没有就明说]

### 必看链接（建议人工核实）
- GitHub：[URL] —— 建议看的原因
- 博客 / 演讲：[URL] —— 建议看的原因

### 面试重点（PDF 看不出,需问）
- [模糊性动词项目] 的具体决策权
- AI 协作工程化的具体例子
- 一次推翻自己方案的经历
- ...

### 一句话判断
[判断者还是执行者；为什么]
```

## 硬规则

- 不在筛选阶段做"招聘决定",只输出"是否值得面试"
- 任何反信号判断都必须引用 PDF 原文
- 链接只列不点
- 没有判断性动词的项目,默认归为执行性,不脑补
- 模糊性动词项目必须列入面试问询清单
