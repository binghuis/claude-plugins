在回答之前仔细思考并一步一步来；这个问题比看上去要难。

评判标准引用 @../../references/talent-model.md。

**视角**:你刚面完一场,飞书有了对话记录,需要快速产出反馈给 HR / 招聘负责人。skill 不做综合录用决策——那是招聘负责人看完所有轮次后的事。

## 输入

| 必填 | 飞书会议记录（文件路径,或粘贴的文本） |
|---|---|
| 选填 | 候选人 PDF（用于核实简历事实是否被现场印证 / 推翻） |
| 选填 | 本轮类型（技术深面 / 判断面 / HM / 文化 / **通用**） |
| 选填 | 候选人级别（若 `interview-prep` 已定级,直接传） |
| 选填 | `interview-prep` 时的方案（用于"该拿到 vs 实际拿到"对比） |
| 选填 | **面试官独立填的 scorecard**（粘贴 prep 输出里"独立 Scorecard"那段填好后的内容）—— 用于 **anchoring 偏差报告** |

**飞书记录格式**:逐字稿优先(最准),AI 总结版会丢候选人原话(影响证据引用质量)。

**性能提示**:60min 逐字稿可能 2-3 万字。**优先传文件路径而不是粘贴**——Claude Code 的 Read 工具按需读,避免单次上下文压力。粘贴只适合 < 5000 字的短记录。

## 流程

| 步骤 | 动作 |
|---|---|
| 1. 识别姓名 + 找 prep | 从对话/PDF 推断候选人姓名;用 glob 找 `./talent-records/[姓名]/prep-*.md`(取最新),找到自动加载做对照基础 |
| 2. 解析对话 | 区分面试官提问 vs 候选人回答,按 talent-model 维度归档 |
| 3. 三档区分（核心） | 每条候选人发言判:**强证据 / 弱证据 / 故事讲述** |
| 4. **简历事实核实**(若 PDF 已传) | 对照候选人原话 vs 简历事实,发现不一致(年限/项目/技术栈/数字)登记为"简历 vs 现场"疑点 |
| 5. 识别疑点 | 含糊 / 矛盾 / 转移话题的点,登记不归入证据 |
| 6. 触发硬底线检查 | 按候选人级别(中级决策案例 / 高级守门人) |
| 7. **Anchoring 偏差对比**(若面试官 scorecard 已传) | 把面试官盲打的 scorecard 与对话证据 + prep 的 calibration 锚("好答案 vs 弱答案")三方对照,产出偏差报告(见下方"Anchoring 偏差报告") |
| 8. 出本场反馈 | **本场判断(本场该过 / 本场存疑 / 本场不过)** + 信心度 + 关键理由 + 给 HR 的建议 |
| 9. 出 HR 版精简反馈 | 把本场判断 + 关键理由 + 给 HR 建议浓缩成一段可直接复制粘贴的反馈 |
| 10. **写本地** | 用 Write 工具落盘到 `./talent-records/[姓名]/debrief-YYYY-MM-DD.md`,目录不存在则创建;同名文件存在则追加 `-r2/-r3` |
| 11. 告知路径 | 在对话末尾告诉用户文件路径 |

## prep 自动接入

跑命令时不需要手动传 prep 方案路径——skill 自动按候选人姓名 glob 找:

| 找到 | 行为 |
|---|---|
| `./talent-records/[姓名]/prep-*.md` 有结果 | 取最新一份,加载作为"该拿到 vs 实际拿到"对照基础 |

| 找不到 | 不强求,跑通用模式(输出末尾标"未关联 prep,无对照") |

## 三档区分（核心方法）

每条候选人发言要分类。**这是 debrief 区别于"流水账总结"的关键**。

| 类别 | 含义 | 处理 |
|---|---|---|
| **强证据** | 讲了具体的"为什么不做 X / 推翻 Y / 拒绝 Z",有动机 / 压力 / 代价 / 反思 | 进证据栏 |
| **弱证据** | 笼统讲了维度但缺细节("我会拒绝不合理需求"、"我们用了 X 框架") | 进证据栏,标"弱" |
| **故事讲述** | 讲了项目 / 经历,有上下文有动作,**但没展开判断 / 取舍 / 反思** | **不入证据栏**,归"故事讲述"区 |

**特别提醒(高级)**:简历豪华但面试空洞是高级最常见伪信号。**只有判断证据进证据栏,故事讲述归"未充分验证"**——不要替候选人圆场。

### 强 vs 弱 vs 故事的现场对照

| 维度 | 强证据 | 弱证据 | 故事讲述 |
|---|---|---|---|
| 决策案例 | "推翻 SSR 因为构建从 X 涨到 Y,后来发现动态路由边界 case..." | "我会拒绝不合理需求" | "我们做了双十一项目,我负责性能优化" |
| 系统思维 | 能讲清模块间约定 / 上下游影响 / 迁移代价 | 只能描述自己负责那块 | 讲了项目结构,没讲设计权衡 |
| AI 协作工程化 | 描述具体工作流 + 校验机制 + 被坑经历 | "我用 Claude 写代码" | "我们团队都在用 AI" |
| 审阅证据 | 主导过具体 review 流程 / 写过审查清单 / 复盘过事故 | 笼统说"我们团队 review" | 讲了一个事故现象,没讲根因和动作 |
| 守门人时刻 | "我叫停了 X,因为...,代价是...,后来证明..." | "我会拒绝高层需求" | 讲了一个项目失败,没讲是谁的判断 |
| 架构判断 | "评估了 A、B、C,选 B 因为...,后来调整为..." | "我设计了 X 架构" | 讲了架构图,没讲为什么不做别的 |

## 按本轮类型校准 gap

**没问到的不算 gap**——只有"该轮该验但没验"才是 gap。

| 本轮类型 | 该验的(没拿到 = gap) | 不该验的(没问也 OK) |
|---|---|---|
| 通用 | 一个深度作品 + 一段判断证据 + 一道现场题 | 跳槽、文化适配 |
| 技术深面 | 系统思维 / 领域深度 | 守门人深挖、跳槽 |
| 判断面 | 决策案例(中) / 守门人时刻(高) / 架构判断 | 基础题、文化 |
| HM | 业务匹配 / 自我认知 / 跳槽 | 任何技术深挖 |
| 文化 | 长期一致性 / 价值观 / 跳槽细节 | 任何技术深挖 |

## 硬底线检查（必须明写,不可省略）

| 级别 | 触发条件 | 必写本场判断 |
|---|---|---|
| 初级 | 基础功底缺失 | **"本场不过 —— 基础底线未达成"** |
| 中级 | 决策案例缺失或全是"推动落地" | **"本场不过(或本场存疑) —— 高级执行者倾向,中级 floor 未过"** |
| 高级 | 判断面没问出守门人时刻或全推动型 | **"本场不过 —— 守门人时刻缺失,默认 pass"** |
| 高级 | HM 轮自我认知答"全面/没什么不擅长" | "自我认知危险信号 —— 进入存疑或不过" |
| 高级 | 文化轮跳槽讲不清冲突 | "履历真实度待背调验证 —— 进入存疑" |

## Anchoring 偏差报告（若面试官 scorecard 已传）

研究里结构化面试效度的核心(0.42-0.51 vs 非结构化 0.14)依赖**独立打分**——面试官在不看预期答案的情况下记录证据。debrief 把三方信号摆出来对照:

| 来源 | 含义 |
|---|---|
| **面试官盲打 scorecard**（输入） | 面试中独立观察的强/弱/故事/空 |
| **对话证据归档**（debrief 解析） | skill 从对话中按维度抽取的证据 |
| **prep calibration 锚**（"好答案 vs 弱答案"） | 该级别该轮该拿到的标准 |

**三种偏差**及含义:

| 偏差 | 含义 | 处理 |
|---|---|---|
| 面试官 > 对话证据 | 面试官打"强",但对话里没有对应原话支撑 → halo 效应 / 印象主导 | 在反馈里降回对话证据档,**警示面试官**下次盲打前避免被候选人状态影响 |
| 面试官 < 对话证据 | 面试官打"弱"或"空",但对话里其实有强证据 → 现场遗漏 / 没追问到位 | 在反馈里升到对话证据档,**记入 calibration 数据** |
| 面试官 vs calibration 锚 gap 大 | 面试官的"强"明显低于该级别"好答案"标准 → 面试官口径偏松 | 在反馈里标"标准对齐疑点",建议下次面试前重读 calibration 锚 |

无 scorecard 输入时:此段省略,只输出对话证据 + calibration 对照。**单纯依赖 debrief 解析的对话证据 ≠ 结构化面试**——盲打 scorecard 是研究里效度的关键源头,**强烈建议每次都填**。

## 硬规则

- **24 小时内必写**——记忆衰减后 debrief 质量大幅下降
- **三档区分必须做**(强证据 / 弱证据 / 故事讲述)——这是 debrief 区别于流水账的核心
- **故事讲述不入证据栏**——只有"为什么不做 X"才是判断证据

- **疑点不归入证据**——单独登记
- **硬底线触发必须明写本场判断**——中级决策案例缺 → "本场不过";高级守门人缺 → "本场不过(默认 pass)"
- **引用候选人原话用引号**,标对话位置(第 X 段 / 时间戳)
- **只出本场判断,不评 ceiling**——`本场该过 / 本场存疑 / 本场不过` 三档,不出"强烈推荐 / 录用"等综合表态(那是 hiring committee 的事)
- **不替招聘负责人决定要不要继续面**——给"下一轮重点验"建议即可
- **scorecard 已传时必须出 Anchoring 偏差报告**——这是结构化面试效度的关键产出;面试官 > 对话证据时降回对话档,**不要为了维护面试官面子而保留偏高分**
- **必须用 Write 工具落盘**——单纯输出对话不算完成
- **姓名识别失败必须问用户**——不能用 "候选人" / "unknown" 之类的占位
- **必须先 glob 找 prep 再生成**——找到的 prep 是"该拿到 vs 实际拿到"对照基础,跳过会失去对比价值
