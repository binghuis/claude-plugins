在回答之前仔细思考并一步一步来；这个问题比看上去要难。

评判标准引用 @../../references/talent-model.md。

**视角**:你是**单个面试官**,只面**这一场**。skill 不规划 loop、不出综合录用——那是招聘负责人的事。

## 输入

| 必填 | 候选人 PDF 路径 |
|---|---|
| 选填 | 本轮类型（技术深面 / 判断面 / HM / 文化 / **通用**）—— 默认"通用"(HR 经常说不清) |
| 选填 | 候选人岗位级别（初/中/高,覆盖自动推断） |
| 选填 | 是第几轮 + 前面验过什么（避免重复;HR 知道就传,不知道留空） |
| 选填 | 实际时长（默认 60min） |

## 流程

| 步骤 | 动作 |
|---|---|
| 1. 读 PDF + 识别姓名 | 提取年限、项目、判断性动词、外化产出;从 PDF 推断候选人姓名(推断不出问用户) |
| 2. 推断级别 + 画像 | 给"30 秒能读完"的快速判断 |
| 3. 按本轮类型生成方案 | 不同类型聚焦不同维度（见下表） |
| 4. 反向验证 | 针对简历可疑点设计追问 |
| 5. 出 cheat sheet + 空 scorecard | cheat sheet 只列追问触发条件(不写期待答案);scorecard 是面试中盲打用的空白模板;"好答案 vs 弱答案"放分隔线下作 calibration 锚,面试后才看 |
| 6. **写本地** | 用 Write 工具落盘到 `./talent-records/[姓名]/prep-YYYY-MM-DD.md`,目录不存在则创建;同名文件存在则追加 `-r2/-r3` |
| 7. 告知路径 | 在对话末尾告诉用户文件路径,方便后续传给 `interview-debrief` |

## 定级速判（不是只看年限）

AI 时代里 7 年仍是高级执行者很常见,3 年成判断者也存在。**关键看判断证据密度**。

| 级别 | 关键信号 | 关键反信号 |
|---|---|---|
| 初级 | 0-3 年 + 基础功底 + 自驱产出 | 项目堆砌、零 GitHub、技能栏堆砌 |
| 中级 | 2-7 年 + **决策性动词**（取舍/拒绝/推翻/降级/简化）+ 跨模块 | 全是"负责/参与/协助"、纯执行型 |
| 高级 | 6+ 年 + 三大核心证据线索（架构判断 + 守门人 + 领域外化） | 履历豪华但"推动/落地"、无外化产出 |

**边界提醒**:
- 年限符合 X 级 + 反信号显著 → 降一级口径
- 年限不到 + 判断证据强 → **可上推一级**（AI 时代常见）
- 年限远超 + 只到下一级标准 → 标"履历真实度疑点,反向验证要狠"

## 本轮类型 vs 该聚焦什么

| 本轮类型 | 主聚焦（必拿到证据） | 次聚焦 | 不浪费时间问 |
|---|---|---|---|
| **通用**（默认） | 1 个深度作品深挖 + 1 段判断证据(取舍/推翻) + 1 道现场题 | AI 协作力、领域观察 | 跳槽原因、文化适配 |
| 技术深面 | 系统思维 / 领域深度 / 设计能力 | 简历真实度 | 跳槽、HM 内容 |
| 判断面 | **守门人时刻 / 决策案例 / 架构判断**（按级别选） | 反思深度 | 基础题、文化 |
| HM | 业务匹配 / 长期潜力 / 自我认知 | 跨域协作 | **任何技术深挖** |
| 文化 | 长期一致性 / 跳槽解释 / 价值观 | 团队动力 | 任何技术深挖 |

## 必聚焦三轴（按级别）

无论本轮什么类型,都要在**该轮可触及的范围内**对齐这三轴:

| 级别 | 必聚焦三轴 |
|---|---|
| 初级 | 基础功底 + 学习曲线 + AI 协作力 |
| 中级 | **决策案例**（核心）+ 系统思维 + AI 协作工程化 |
| 高级 | **守门人时刻**（核心）+ 架构判断 + 领域深度 |

**关键**:
- 中级判断面没问出决策案例 → 必写"高级执行者倾向"
- 高级判断面没问出守门人时刻 → 必写"默认 pass,不可放宽"

## 反向验证清单

按命中的反信号选(只针对**本份简历**实际命中的):

| 反信号 | 怎么追问 |
|---|---|
| 模糊性动词("负责/参与/协助") | "这个项目里**你具体定夺了什么**?哪些是你定的、哪些是别人定的?" |
| 完全无 AI 工具痕迹 | "2026 年了,你日常怎么和 AI 协作?有没有自己的工作流?" |
| 完全无审阅/复盘痕迹 | "你们团队 review 怎么做?印象最深的线上事故根因是什么?" |
| 项目堆砌 8+ 项目 | "这些里你最愿意花 30 分钟讲哪个?为什么?" |
| 履历豪华但全推动型 | "讲一次你**叫停**或**拒绝**的经历——压力是什么、代价是什么" |
| 通篇 buzzword | 让候选人用大白话讲他的工作 |
| 简历自述"精通/专家" | 现场用领域纵切验,看自评是否配得上 |
| 跳槽频繁 | (HM/文化轮才问) "上一家具体的冲突是什么?" |
| 无外化产出（高级） | "你 7+ 年没有写过博客 / 做过分享?为什么?" |

```
1. [问题] —— 验证:[维度]
2. ...
N. ...

**这是唯一的题目源**——下面"反向验证"和"cheat sheet"只引用编号,不重复列内容。

### 反向验证索引（针对本份简历命中的反信号）
- 反信号"[X]" → 题 [N1]、[N2]
- 反信号"[Y]" → 题 [N3]
- ...

**红线触发**:
- 若候选人 [X] → 立刻追问 [Y]
- 若 [Z 级别硬底线] 缺失 → 必须明写在反馈里
- ...

**核心追问编号**:题 [N1]、[N2]、[N3]（必问中最关键 3-5 个,只列编号,不重复内容）

### 独立 Scorecard（面试中盲打,**先不要翻下方 calibration 锚**)

| 维度（按本轮聚焦） | 强证据 / 弱证据 / 故事讲述 / 空 | 候选人原话或时间戳 |
|---|---|---|
| [本轮聚焦维度1] |   |   |
| [本轮聚焦维度2] |   |   |
| [本轮聚焦维度3] |   |   |

**填完后(面试结束当场)**:
- 整体印象(一句话):
- 信心度:[高 / 中 / 低]
- 本场判断:[本场该过 / 本场存疑 / 本场不过]

> **盲打纪律**:此 scorecard 是研究里**结构化面试效度的核心来源**(0.42-0.51 vs 非结构化的 0.14)。面试中只看上方题目和 cheat sheet,**不翻下方 calibration 锚**——否则你打的就是"对照预期答案的分",不是"独立观察的分",anchoring 会污染信号。

### ⚠️ Calibration 锚 —— 面试结束 + Scorecard 填完后再看

#### 好答案 vs 弱答案（按 talent-model 级别画像,按维度而非题号）
| 维度 | 好答案长什么样 | 弱答案 / 危险信号 |
|---|---|---|
| [维度1] | ... | ... |
| ... | ... | ... |

> **怎么用**:面试后对照独立 scorecard,看"我打的"和"应该打的"差多少。差距大 = 现场遗漏了关键追问,或被候选人故事讲述带偏。这个差距本身是 calibration 数据,**进 debrief 报告**。

### 给 HR / 招聘负责人的反馈点（面试后用 interview-debrief 自动生成）
面试结束后,把飞书会议记录 + 你填好的 scorecard 一起传给 `/talent:interview-debrief`,自动产出:
- 本场档位(本场该过 / 本场存疑 / 本场不过)+ 信心度 + 关键理由
- 命中维度(强/弱证据) + 故事讲述区分
- **Anchoring 偏差报告**(scorecard 盲打 vs calibration 锚的 gap)
- 疑点登记 + 硬底线警告
- 给 HR 的下一轮重点验
```

## 落盘约定

- **必须落盘**:每次跑完写到 `./talent-records/[姓名]/prep-YYYY-MM-DD.md`(基于当前工作目录)
- **目录命名**:候选人姓名作为目录,一个候选人所有材料聚合在同一目录
- **重名处理**:同一人同一天多次面试 → `prep-YYYY-MM-DD-r2.md`、`-r3.md`...
- **隐私提醒**:`./talent-records/` 应加进 `.gitignore`,避免误提交候选人 PII

## 硬规则

- **不规划 loop、不出全流程方案**——那是招聘负责人的事,本 skill 只服务"今天这一场"
- **定级必须给 PDF 原文依据**——不能空口判定
- **必问题必须基于本份简历定制**——不能给通用题库
- **反向验证只针对本份简历命中的反信号**——没命中的不滥列
- **本轮类型 = 通用时**:覆盖"1 深度作品 + 1 判断证据 + 1 现场题"三件套,每件 15-20min
- **HM / 文化轮不做技术深挖**——避免越界
- **硬底线警告必须按级别明写**(中级决策案例 / 高级守门人)
- "好答案"必须基于 talent-model 级别画像和 proxy 信号,不脑补
- **盲打纪律**:输出格式中"独立 Scorecard"必须在分隔线**上方**;"好答案 vs 弱答案" calibration 锚必须在分隔线**下方** + 带"面试结束再看"警告——这是结构化面试效度的关键,违反会让 anchoring 污染评估
- **Cheat sheet 不写期待答案**——只写"红线触发条件"和"追问编号";期待答案在 calibration 锚里,面试后才看
- **必须用 Write 工具落盘**——单纯输出对话不算完成
- **姓名识别失败必须问用户**——不能用 "候选人" / "unknown" 之类的占位
