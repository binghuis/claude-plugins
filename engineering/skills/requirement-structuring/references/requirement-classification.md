# 需求判定

## 判据

- 需求：功能 / 业务规则 / 逻辑分支
- 非需求：视觉 / 交互 / 技术
- 混合：业务进需求清单，其余各归本位

## 示例

| 原始信息 | 归类 |
|---|---|
| 支持手机号登录 | 需求（功能） |
| 验证码 6 位、5 分钟有效、错 3 次锁定 | 需求（规则） |
| 未注册手机号引导去注册 | 需求（逻辑分支） |
| 登录按钮蓝色、"好看""高级感" | 非需求（视觉） |
| loading 动画、"丝滑""流畅" | 非需求（交互） |
| 用 Redis、"用 XX 框架" | 非需求（技术，除非"兼容老系统"等业务约束） |
| 网络超时重试、提示"网络异常" | 重试→需求（分支）；文案→非需求（视觉） |
