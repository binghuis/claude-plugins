# 需求识别

- 需求：功能 / 业务规则 / 逻辑分支
- 非需求：视觉 / 交互 / 技术
- 混合素材：业务部分入需求清单，其余各归本位

UI 图传达的是意图，不是实现细节。

| 原始素材 | 归类 |
|---|---|
| 支持手机号登录 | 需求（功能） |
| 验证码 6 位、5 分钟有效、错 3 次锁定 | 需求（规则） |
| 未注册手机号引导至注册 | 需求（分支） |
| UI 图小字"30 秒后重新获取" | 需求（暗示规则 → 验证码冷却） |
| 登录按钮要蓝色、要"高级感" | 非需求（视觉） |
| UI 图的颜色、间距、对齐偏差 | 非需求（视觉细节，不入清单也不澄清） |
| 指定用 Redis、用某框架 | 非需求（技术，除非属于"兼容老系统"等业务约束） |
| 登录失败提示"账号或密码错误" | 反馈策略 → 需求（分支）；具体文案 → 非需求（视觉） |
